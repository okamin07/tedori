# TEDORI 要件定義・実行計画

## 1. コンセプト

副業・独立の「手取り」を**シナリオ比較**で決めるツール。

- **ポジション**: 副業・独立の「もしも」比較器
- **MVPの約束**: Geminiに聞くより、**同じ入力で同じ数字・2シナリオの差分**が見える

### MVPでやらないこと（初期段階）

| やらない | 理由 |
|---|---|
| ユーザーアカウント | 集客・試算・アフィリが先。ログインは離脱要因 |
| 課金・プラン・マイページ | 収益はアフィリ。課金は保存・PDF等が必要になってから |
| SaaS型サイドバー | ツール1本＋SEO記事が主役。ダッシュボード感は不要 |

## 2. 集客・SEO戦略

### 2.1 キーワード設計（3層）

**Layer A — ツール意図（CV直結）**

| キーワード | 月間想定競合 | 対応 |
|---|---|---|
| 副業 手取り 計算 | 高 | `/` ツール + 記事 `fukugyo-tedori-simulator` |

| 副業 税金 シミュレーション | 中 | ツール H1・meta |
| 個人事業主 手取り シミュレーター | 中 | 独立モード・記事 |

| キーワード | 対応記事 | ASP |

### 2.2 検索意図と記事型

| Intent | 記事の型 | 例 |
|---|---|---|
| Know | 結論→理由→具体例→FAQ | 副業 税金 いくら |
| Do | 手順 + ツールCTA | 手取り 計算 |
| Compare | 2案の差分表 + ツール | 青 vs 白 |
| Commercial | 比較表 + 体験談風 + CTA | 会計ソフト |

### 2.3 上位表示のための必須要素

1. **タイトルに主キーワード + 年号**（2025年）
2. **最初の段落で結論**（Featured snippet 狙い）
3. **FAQ構造化データ**（FAQPage JSON-LD）
4. **Article + Breadcrumb** JSON-LD
5. **内部リンク**: 記事 ↔ ツール ↔ 関連記事（relatedSlugs）
6. **具体数字**（例: 本業500万・副業50万）
7. **読了時間 7分+** — 薄い3段落記事は勝てない

### 2.4 サイト構造（IA）

```
/                 手取り比較ツール（メインCV）
/media            記事ハブ（SEO流入）
/media/[slug]     ロングテール各記事
```

アカウント・設定・課金ページなし。

## 3. アフィリエイト戦略

### 3.1 収益モデル（初期）

**100% アフィリエイト**。試算結果 → 文脈付きCTA1本。

| タイミング | 提案 | プログラム |
|---|---|---|
| 確定申告必要ライン | 記帳・申告 | 会計ソフト |
| 青色の方が有利（差5万+） | 開業・青申請 | 開業freee等 |
| 実効税率25%+ | 経費計上 | 会計ソフト |
| 所得500万+ | 税理士 | 紹介ASP |
| 経費記事末尾 | 事業用カード | カードASP |

### 3.2 CTA原則

- 記事末の汎用バナー禁止 → **試算数字に紐づく1文**
- 商用記事は比較表 + 無料トライアル訴求
- `rel="sponsored nofollow"` 厳守

---

## 4. ツール要件

- モード: 副業（会社員） / 独立（個人事業）
- 比較: 現在の申告 vs 反対側（青↔白）
- 出力: 年間差額、2カード、内訳バー、差分表
- 入力: 同一ページ内（別URL不要）
- 計算: 2025年分概算、pure関数+テスト

## 6. デザイン方針

- **ツールファースト**: ヘッダー2リンク（比較 / 読みもの）のみ
- 比較UIはワイヤー準拠（差額ヒーロー → 2カード → 内訳 → 表）
- 記事は3列カード + 検索 + カテゴリフィルタ
- SaaSサイドバー・プラン・ゲストユーザー表示なし
